# GPL違規：閉源安卓內核

> 不是「他們可能違規了」，而是「他們確實沒遵守 GPL」。

---

## 先說一下GPL是什麼

GPL（GNU General Public License）是一個開源許可證。簡單說就是：

- 你可以免費使用基於GPL協議的程式碼
- **但如果你修改了程式碼並分發產品，必須把修改後的原始碼也公開**
- Linux 內核用的是 GPLv2，這是硬性要求，不是可以商量的

小天才的手錶跑的是安卓系統，而安卓系統底層就是 Linux 內核。這意味著只要你用了 Linux 內核做了產品並賣出去，你就必須把內核原始碼（包括你修改過的部份）提供出來。

這不是什麼灰色地帶。Google 自己都老老實實把 Pixel 的內核原始碼放到了 [kernel.google.com](https://kernel.google.com) 上。三星發了，小米發了（雖然慢），OPPO 也發了。

## 小天才沒發

截至這個項目存在的這一天，小天才沒有在任何一個公開渠道釋出過任何一款裝置的內核原始碼。

你去他們官網找，找不到。
你去 GitHub 搜「xiaotiancai」或者「imoo kernel」，搜不到。
你去內核郵件列表裡翻，也沒有。

這不是「找不到」的問題——是他們根本就沒發。

## 為什麼說這是違規

GPLv2 的條款寫得清清楚楚：

> If you distribute copies of such a program, you must also make the source code available.

你把裝了 Linux 內核的手錶賣出去了，那就是「distribute copies」，你就必須提供原始碼。沒有例外。

有些公司會辯解說是「用的是沒改過的內核」，那你也得提供原始碼。更何況小天才的手錶肯定是改過的——他們要適配展銳或者其他方案的SoC，要做驅動適配，要做各種硬體調教，不可能不改。

## 為什麼不發的公司這麼多

這是個現實問題。在國內做安卓裝置的廠商裡，不守 GPL 的遠不止小天才一家。很多小廠、白牌裝置都不發。但這不代表這是對的。

只不過以前沒人盯著兒童手錶這個品類而已。

## 這有什麼實際影響

內核原始碼不發，意味著：

1. **安全研究沒法做。** 有沒有後門？有沒有漏洞？沒人能審計
2. **第三方 ROM 不可能。** 原廠系統爛你也得忍著
3. **裝置生命週期完全由廠商掌控。** 廠商說不更新了，你的裝置就死了
4. **想自己修 bug？** 門都沒有

一台給孩子用的裝置，黑箱狀態，不允許任何人審查它的底層程式碼。

你覺得這合適嗎？
