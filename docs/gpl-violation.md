# GPL违规：闭源安卓内核

> 不是"他们可能违规了"，而是"他们确实没遵守 GPL"。

---

## 先说一下GPL是什么

GPL（GNU General Public License）是一个开源许可证。简单说就是：

- 你可以免费使用基于GPL协议的代码
- **但如果你修改了代码并分发产品，必须把修改后的源码也公开**
- Linux 内核用的是 GPLv2，这是硬性要求，不是可以商量的

小天才的手表跑的是安卓系统，而安卓系统底层就是 Linux 内核。这意味着只要你用了 Linux 内核做了产品并卖出去，你就必须把内核源码（包括你修改过的部分）提供出来。

这不是什么灰色地带。Google 自己都老老实实把 Pixel 的内核源码放到了 [kernel.google.com](https://kernel.google.com) 上。三星发了，小米发了（虽然慢），OPPO 也发了。

## 小天才没发

截至这个项目存在的这一天，小天才没有在任何一个公开渠道发布过任何一款设备的内核源码。

你去他们官网找，找不到。
你去 GitHub 搜 "xiaotiancai" 或者 "imoo kernel"，搜不到。
你去内核邮件列表里翻，也没有。

这不是"找不到"的问题——是他们压根就没发。

## 为什么说这是违规

GPLv2 的条款写得清清楚楚：

> If you distribute copies of such a program, you must also make the source code available.

你把装了 Linux 内核的手表卖出去了，那就是"distribute copies"，你就必须提供源码。没有例外。

有些公司会辩解说是"用的是没改过的内核"，那你也得提供源码。更何况小天才的手表肯定是改过的——他们要适配展锐或者其他方案的 SoC，要做驱动适配，要做各种硬件调教，不可能不改。

## 为什么不发的公司这么多

这是个现实问题。在国内做安卓设备的厂商里，不守 GPL 的远不止小天才一家。很多小厂、白牌设备都不发。但这不代表这是对的。

只不过以前没人盯着儿童手表这个品类而已。

## 这有什么实际影响

内核源码不发，意味着：

1. **安全研究没法做。** 有没有后门？有没有漏洞？没人能审计
2. **第三方 ROM 不可能。** 原厂系统烂你也得忍着
3. **设备生命周期完全由厂商掌控。** 厂商说不更新了，你的设备就死了
4. **想自己修 bug？** 门都没有

一台给孩子用的设备，黑箱状态，不允许任何人审查它的底层代码。

你觉得这合适吗？
